# Neo MCP Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `NEO_SECRET_KEY` | **Yes** | API key (`sk-v1-...`) from [Neo dashboard](https://heyneo.com/dashboard?section=settings#access-keys) |
| `NEO_WORKSPACE_DIR` | **Yes (Postman)** | Absolute path to your project/git root where Neo writes files. Set this in Postman Environment so tasks land in the correct folder. |
| `NEO_ENVIRONMENT` | No | `production` (default) or `staging` |
| `NEO_DEPLOYMENT_ID` | No | Pin a specific daemon UUID (auto-generated by default) |
| `NEO_READ_ONLY` | No | `true` = read-only tools only (no submit/stop/pause) |

## Postman Environment

Create variables:

- `NEO_SECRET_KEY` — your `sk-v1-...` key (secret)
- `NEO_WORKSPACE_DIR` — e.g. `/Users/you/projects/my-ml-repo` (absolute path)

Reference them in the MCP STDIO config:

```json
{
  "mcpServers": {
    "neo": {
      "command": "uvx",
      "args": ["neo-mcp"],
      "env": {
        "NEO_SECRET_KEY": "{{NEO_SECRET_KEY}}",
        "NEO_WORKSPACE_DIR": "{{NEO_WORKSPACE_DIR}}"
      }
    }
  }
}
```

## workspace vs NEO_WORKSPACE_DIR

- `NEO_WORKSPACE_DIR` — default workspace for the MCP server process
- `workspace` arg on `neo_submit_task` — per-task override (must still be absolute project root)

For Postman, set `NEO_WORKSPACE_DIR` and pass the same path as `workspace` on submit.
